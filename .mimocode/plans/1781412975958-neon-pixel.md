# 纯语音绘图能力增强 + 设计文档

## 背景
比赛要求："用户不能使用鼠标或键盘，仅通过语音指令完成绘图创作。" 需要聚焦语音识别准确性、响应延迟、指令覆盖度，并提交设计文档。

## 当前问题分析

| 问题 | 严重程度 | 位置 |
|------|----------|------|
| Whisper beam_size=10, best_of=5 导致延迟 2-5 秒 | 高 | core.py:640-641 |
| 确认流程依赖 stdin（按回车），非纯语音 | 高 | core.py:1078-1122 |
| 文字指令 `写xxx` 需要引号，语音难以触发 | 中 | commands.py:1866 |
| 语音反馈只有视觉（屏幕文字），无听觉反馈 | 中 | core.py 全局 |
| `pinyin_replace_unknown()` 是死代码，未被调用 | 低 | pinyin_matcher.py:96 |
| 部分常用语音表达未覆盖（如"画个圈"、"来个红的"） | 中 | commands.py _ZH_EN_MAP |
| 自动保存 PNG 写入阻塞主线程 | 低 | core.py:1224 |

## 修改计划

### 1. 优化 Whisper 参数降低延迟
**文件**: `src/voice_drawing_tool/core.py`

```
beam_size: 10 → 5     (减少搜索宽度，延迟降低约 40%)
best_of: 5 → 3        (减少候选数，延迟降低约 30%)
```

同时保持识别质量：保留 word_timestamps、vad_filter、repetition_penalty。

### 2. 语音确认流程（去除 stdin 依赖）
**文件**: `src/voice_drawing_tool/core.py`

当前流程：语音识别 → 打印预览 → 等 stdin 回车 → 执行
改为：语音识别 → 语音播报确认 → 自动执行

- 删除 `_pending_speech_cmd` 等待 stdin 确认的逻辑
- 语音识别到命令后直接执行（去掉中间确认步骤）
- 保留清空/退出的二次确认（通过语音说"确认"触发）
- `_speech_loop` 中：识别到命令 → 调用 `execute_command()` → 语音播报结果

### 3. 添加语音反馈（pyttsx3）
**文件**: `src/voice_drawing_tool/core.py`

pyttsx3 已安装但未使用。添加 `VoiceFeedback` 类：
- 识别到命令时播报："画了一个红色的圆"
- 执行成功播报："完成"
- 未识别播报："没听清，请再说一次"
- 清空确认播报："请说确认来清空画布"
- 使用独立线程播报，不阻塞主循环

### 5. 文字指令正则优化
**文件**: `src/voice_drawing_tool/commands.py`

当前 `DrawTextCommand` 需要引号：`write 'Hello' at 50,50`
语音无法说出引号。添加无引号的文字指令模式：
```
写文字xxx在100,200
写xxx在100,200
```

在 `_register()` 中添加新 pattern，在 `_ZH_EN_MAP` 中确保 `写` → `write` 映射正确。

### 6. 异步自动保存
**文件**: `src/voice_drawing_tool/core.py`

将 `cv2.imwrite('/tmp/voice_drawing_autosave.png', ...)` 改为异步：
- 使用独立线程保存，不阻塞主循环
- 或改为每 5 次命令保存一次（节流）

### 7. 设计文档
**新增文件**: `DESIGN.md`

内容结构：
1. 项目概述与目标
2. 支持的指令能力清单（分类列出）
3. 语音识别架构（Whisper + 纠错 + 拼音模糊匹配）
4. 指令解析流水线（语音→纠错→翻译→正则→命令对象）
5. 延迟优化策略
6. 已实现 vs 计划中 vs 未完成（含原因）
7. 创新点与技术亮点
